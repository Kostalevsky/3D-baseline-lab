# **Методология обработки 3D-объектов**

---

## **ISO-ориентированный метод ортографических проекций**

В качестве первого базового подхода для подготовки визуального представления 3D-объекта был рассмотрен ISO-ориентированный метод ортографических проекций. В отличие от адаптивных методов, которые выбирают наиболее информативные ракурсы на основе геометрии объекта или с помощью модели, данный подход использует заранее заданную инженерную схему представления формы. Объект переводится в набор стандартных видов: спереди, сверху, снизу, слева, справа и сзади. Эти виды формируют технический коллаж, который может быть использован как вход для последующей работы мультимодальной языковой модели.

Метод основан на логике ортографического проецирования, описанной в стандарте ISO 5456-2. В таком представлении 3D-объект отображается на плоскости без перспективных искажений, поэтому каждый вид сохраняет геометрические пропорции соответствующей стороны объекта. В зависимости от выбранной системы расположения видов могут использоваться first-angle projection или third-angle projection. В third-angle projection верхний вид располагается над главным видом, нижний — под ним, левый — слева, правый — справа. В first-angle projection расположение противоположное: верхний вид находится под главным видом, нижний — над ним, левый вид — справа, а правый — слева.

Для задачи генерации коллажа этот подход можно рассматривать как воспроизводимый baseline. Он не требует обучения модели, не зависит от семантики объекта и может быть применён к разным форматам 3D-данных, например OBJ, STL, PLY или GLB. На вход программа получает 3D-модель, нормализует её по центру и масштабу, после чего рендерит шесть ортографических проекций с фиксированных направлений камеры. Далее изображения размещаются в коллаже согласно выбранной схеме: first-angle или third-angle. Такой коллаж позволяет представить объект в форме, близкой к техническому чертежу, и делает структуру объекта более понятной для дальнейшего анализа.

Преимущество ISO-подхода заключается в его простоте, стандартизированности и интерпретируемости. Он хорошо подходит для инженерных и CAD-объектов, где важны форма, пропорции, симметрия, отверстия, выступы и взаимное расположение элементов. Кроме того, этот метод удобен как контрольная точка для сравнения с более сложными подходами: алгоритмическим выбором информативных ракурсов, multi-view методами и LLM/VLM-based view selection.

Ограничение метода состоит в том, что он не выбирает ракурсы адаптивно. Для некоторых объектов часть стандартных видов может быть малоинформативной или дублировать уже видимые признаки. Кроме того, ортографические проекции не всегда хорошо передают объёмную структуру органических или сложных объектов. Поэтому ISO-подход целесообразно использовать как базовый технический способ генерации коллажа, а затем сравнивать его с методами, которые автоматически оценивают информативность ракурсов.

## **Cap3D-style метод многоракурсного представления объекта**

В качестве второго подхода был рассмотрен Cap3D-style метод, основанный на представлении 3D-объекта через набор изображений, полученных с разных ракурсов. В отличие от ISO-ориентированного подхода, где используются фиксированные ортографические виды, данный метод стремится показать объект более естественно для мультимодальной модели: не как технический чертёж, а как серию визуальных наблюдений вокруг объекта.

Оригинальный подход Cap3D направлен на генерацию текстовых описаний 3D-объектов. В общем виде его логика строится следующим образом: 3D-объект рендерится с нескольких ракурсов, затем для полученных изображений генерируются текстовые описания с помощью image captioning моделей, после чего результаты фильтруются, сопоставляются и объединяются в итоговое описание объекта при помощи языковой модели. Таким образом, Cap3D использует 2D-представления 3D-модели как промежуточный слой между геометрией объекта и языковым описанием.

В рамках текущей реализации был воспроизведён не полный Cap3D pipeline, а его визуальный preprocessing-этап. Программа получает 3D-модель, нормализует её по центру и масштабу, после чего рендерит восемь изображений объекта с равномерно распределённых ракурсов вокруг вертикальной оси. Камера движется по окружности вокруг модели, сохраняя одинаковую высоту и расстояние до центра объекта. Полученные изображения затем объединяются в коллаж, который может быть использован как вход для дальнейшей обработки мультимодальной моделью.

Такой подход можно описать как:
```
3D object
    ↓
mesh normalization
    ↓
render 8 views around the object
    ↓
assemble multi-view collage
    ↓
save collage + metadata
```

В отличие от ISO-подхода, Cap3D-style baseline использует перспективную камеру и диагональные ракурсы. Это позволяет лучше передавать объёмную форму объекта, взаимное расположение частей и общий внешний вид. Такой формат ближе к тому, как объект воспринимается человеком или visual-language моделью при просмотре набора изображений.

Преимущество Cap3D-style подхода заключается в том, что он лучше подходит для визуального понимания формы в задачах, где важны не только отдельные стороны объекта, но и его общий силуэт, объём и характерные детали. Он особенно полезен для объектов, которые трудно описать только через front/top/side views. Кроме того, такой baseline хорошо связан с дальнейшей LLM/VLM-логикой: на следующем этапе к каждому виду можно добавить автоматическое captioning-описание, а затем объединить эти описания в итоговое описание 3D-объекта.

Ограничение текущей реализации состоит в том, что она пока не включает captioning-модуль и не выполняет полноценное текстовое описание объекта. Реализован только этап генерации визуального представления: рендеринг нескольких видов и сборка коллажа. Кроме того, ракурсы выбираются фиксированно и равномерно, без оценки их информативности. Поэтому данный подход следует рассматривать как промежуточный baseline между строгим ISO-представлением и более сложными методами, где ракурсы выбираются адаптивно или с помощью мультимодальной модели.

## **Алгоритмический метод выбора информативных ракурсов**

В качестве третьего подхода был рассмотрен алгоритмический метод выбора информативных ракурсов. В отличие от ISO-подхода, где набор видов задаётся инженерным стандартом, и Cap3D-style подхода, где используется фиксированное количество равномерно распределённых ракурсов, данный метод пытается автоматически определить, какие изображения объекта являются наиболее полезными для дальнейшего анализа.

Общая логика метода состоит в том, что программа сначала рендерит расширенный набор candidate views. В текущей реализации используются 24 кандидатных ракурса: восемь углов азимута вокруг объекта и три уровня высоты камеры. После этого каждый rendered view оценивается с помощью простых визуальных признаков, которые можно посчитать без обучения модели.

Для оценки информативности используются следующие признаки:

```text
1. Silhouette area — доля изображения, занятая объектом.
2. Bounding box fill ratio — насколько плотно объект заполняет свой 2D bounding box.
3. Edge / contour complexity — количество контурных переходов в изображении.
4. View diversity — отличие выбранного ракурса от уже отобранных видов.
```

Silhouette area помогает отсеивать ракурсы, где объект занимает слишком мало места в кадре. Bounding box fill ratio показывает, насколько компактно объект представлен в изображении. Edge / contour complexity полезен для объектов с отверстиями, вырезами и сложной внешней границей: чем больше контуров видно, тем потенциально информативнее вид. View diversity добавляется на этапе выбора top views, чтобы итоговый коллаж не состоял из почти одинаковых ракурсов.

В текущей реализации итоговый score считается как взвешенная сумма:

```text
final_score =
    0.45 * area_score
  + 0.35 * edge_score
  + 0.20 * bbox_fill_score
```

После расчёта score программа выбирает восемь лучших ракурсов. Выбор выполняется greedy-алгоритмом: первый вид выбирается по максимальному score, а следующие получают дополнительный бонус за угловое отличие от уже выбранных ракурсов. Это позволяет сохранить баланс между информативностью и разнообразием итогового коллажа.

Такой подход можно описать как:

```text
3D object
    ↓
mesh normalization
    ↓
render 24 candidate views
    ↓
calculate visual score for each view
    ↓
select top 8 diverse views
    ↓
assemble collage
    ↓
save collage + metadata
```

Преимущество алгоритмического подхода заключается в том, что он не требует обучения модели и остаётся полностью интерпретируемым. В metadata сохраняются все оценки, поэтому можно понять, почему конкретные ракурсы были выбраны. Это делает метод удобным baseline для сравнения с LLM/VLM-based view selection, где выбор ракурсов может быть менее прозрачным.

На плоских инженерных деталях такой метод особенно полезен: чисто боковые виды часто оказываются малоинформативными, потому что объект выглядит как тонкая полоска. Алгоритмический baseline, напротив, выбирает верхние и диагональные ракурсы, где видны отверстия, силуэт, толщина и пространственная форма детали.

Ограничение метода состоит в том, что он оценивает только визуальные признаки rendered image и не понимает семантику объекта. Например, ракурс может быть важен с инженерной точки зрения, даже если его silhouette area или contour complexity невысоки. Поэтому данный подход следует рассматривать как объяснимый geometric/visual baseline, а не как полноценную замену семантическому выбору ракурсов с помощью VLM.

## Описание реализованной программы

## Описание реализованной программы

Для проверки рассмотренных методов была начата разработка экспериментальной Python-системы, позволяющей загружать 3D-объект, выбирать baseline-метод обработки и получать на выходе визуальный коллаж с метаданными запуска. Основная идея программы заключается в том, чтобы разные методы подготовки 3D-объектов можно было запускать в одном формате и затем сравнивать их результаты между собой.

На текущем этапе система поддерживает три baseline-метода:

```text
1. ISO Orthographic Baseline
2. Cap3D-style Views Baseline
3. Algorithmic Top Views Baseline
```

Каждый baseline реализует собственную стратегию выбора и рендеринга ракурсов. ISO-подход использует стандартные ортографические проекции, Cap3D-style baseline создаёт фиксированные восемь перспективных видов вокруг объекта, а Algorithmic Top Views baseline сначала генерирует расширенный набор кандидатных ракурсов, оценивает их по визуальным признакам и выбирает восемь наиболее информативных видов.

```text
input 3D model / demo object
        ↓
load mesh
        ↓
normalize mesh
        ↓
run selected baseline
        ↓
render views
        ↓
assemble collage
        ↓
save output image
        ↓
save metadata JSON
```

Программа написана на Python и использует несколько основных библиотек. trimesh применяется для загрузки и базовой обработки 3D-моделей, pyrender — для рендеринга объекта в изображения, Pillow — для сборки финального коллажа, а numpy — для работы с координатами камеры, векторами и расчётом визуальных признаков.

Важная особенность системы — единый формат входа и выхода. Независимо от выбранного baseline, программа принимает 3D-модель, нормализует её, запускает выбранный метод и сохраняет результат в виде collage.png и metadata.json. Это позволяет постепенно добавлять новые подходы и сравнивать их между собой в одной структуре проекта.

## Загрузка и нормализация модели

Перед запуском любого baseline программа должна привести 3D-модель к единому виду. Это необходимо, потому что разные модели могут иметь разный масштаб, положение в пространстве и количество объектов внутри сцены.

Сначала модель загружается из файла. Поддерживаемые форматы зависят от библиотеки `trimesh`, но обычно это:

```text

.obj

.stl

.ply

.glb

```

Если модель загружается как один mesh, программа использует его напрямую. Если модель загружается как сцена, состоящая из нескольких отдельных geometry-объектов, программа объединяет их в один общий mesh. Это нужно для того, чтобы baseline-методы могли работать с объектом единообразно.

После загрузки выполняется нормализация. Она состоит из двух шагов:

```text

1. Центрирование объекта

2. Масштабирование объекта

```

Центрирование переносит объект так, чтобы его bounding box оказался расположен вокруг начала координат. Это важно, потому что все камеры в программе смотрят в точку `(0, 0, 0)`. Если объект не центрировать, камера может смотреть мимо модели или модель будет находиться не в центре изображения.

Масштабирование приводит объект к единому размеру: максимальная сторона bounding box становится равной `1`. Это позволяет рендерить разные модели с одинаковыми параметрами камеры. Без этого маленькие модели могли бы быть почти невидимыми, а слишком большие — выходить за границы кадра.

## ISO Orthographic Baseline в программе

ISO baseline реализован в файле:

```text

src/baseline_lab/baselines/iso.py

```

Он создаёт шесть ортографических изображений объекта:

```text

front

back

right

left

top

bottom

```

Для каждого вида программа задаёт фиксированное положение камеры. Например, front-view получается камерой, расположенной перед объектом, back-view — камерой позади объекта, top-view — камерой сверху. Все камеры направлены в центр координат, то есть в центр нормализованного объекта.

Главное отличие ISO baseline от Cap3D-style baseline состоит в том, что здесь используется ортографическая камера. Ортографическая камера не создаёт перспективных искажений: удалённые и ближние части объекта не меняют размер в зависимости от расстояния до камеры. Благодаря этому изображение похоже на техническую проекцию.

После рендеринга шести видов программа собирает их в коллаж. Поддерживаются две схемы расположения:

```text

third-angle projection

first-angle projection

```

В third-angle projection виды располагаются интуитивно:

```text

        TOP

LEFT   FRONT   RIGHT   BACK

        BOTTOM

```

В first-angle projection расположение отличается:

```text

        BOTTOM

RIGHT  FRONT   LEFT    BACK

        TOP

```

На выходе ISO baseline возвращает изображение-коллаж и metadata-словарь, где фиксируются параметры запуска: название baseline, тип проекции, список видов и размер каждого изображения.

## Cap3D-style Views Baseline в программе

Cap3D-style baseline реализован в файле:

```text

src/baseline_lab/baselines/cap3d.py

```

Он генерирует восемь ракурсов объекта, равномерно расположенных вокруг вертикальной оси. В отличие от ISO baseline, здесь не используются front/top/side views в формате технического чертежа. Вместо этого камера движется вокруг объекта по окружности.

Текущая схема ракурсов выглядит так:

```text

0°

45°

90°

135°

180°

225°

270°

315°

```

Для каждого ракурса программа вычисляет положение камеры по углу азимута. Высота камеры задаётся параметром `elevation_deg`, а расстояние до объекта — параметром `radius`. По умолчанию используется восемь видов, но архитектурно этот параметр можно изменить.

Общий принцип работы такой:

```text

for each view:

    calculate camera position

    create render scene

    add mesh

    add perspective camera

    add light

    render image

```

В этом baseline используется перспективная камера. Это делает изображения более похожими на обычные визуальные рендеры, где объект воспринимается объёмно. Такой формат полезен для последующей работы VLM/LLM, потому что модель получает несколько визуально разных наблюдений объекта, а не только плоские инженерные проекции.

После рендеринга восьми изображений программа собирает их в коллаж в виде сетки. В текущей реализации используется раскладка `4 × 2`: четыре изображения в верхнем ряду и четыре изображения в нижнем. Каждый tile получает подпись с номером вида и углом камеры, например:

```text

VIEW_00_0DEG

VIEW_01_45DEG

VIEW_02_90DEG

...

VIEW_07_315DEG

```

На выходе Cap3D-style baseline сохраняет коллаж и metadata, где указаны количество ракурсов, угол elevation, радиус камеры, размер изображений и список всех сгенерированных views.

## Algorithmic Top Views Baseline в программе

Algorithmic Top Views baseline реализован в файле:

```text
src/baseline_lab/baselines/algorithmic.py
```

Этот baseline отличается от ISO и Cap3D-style подходов тем, что он не использует заранее фиксированный итоговый набор видов. Вместо этого программа сначала создаёт расширенный набор кандидатных ракурсов, затем оценивает каждый из них по визуальным признакам и выбирает наиболее информативные изображения для итогового коллажа.

В текущей реализации создаётся 24 кандидатных вида:

```text
8 azimuth angles × 3 elevation levels = 24 candidate views
```

Используются следующие уровни высоты камеры:

```text
15°
35°
60°
```

И восемь направлений вокруг объекта:

```text
0°
45°
90°
135°
180°
225°
270°
315°
```

Таким образом, программа получает набор ракурсов, который включает более низкие, средние и более верхние перспективные виды. Это особенно полезно для плоских инженерных деталей: низкие боковые ракурсы часто показывают только толщину объекта, а более высокие виды лучше раскрывают отверстия, силуэт и общую форму.

Общий принцип работы baseline:

```text
for each candidate view:
    calculate camera position
    render image
    create object mask
    calculate visual scores

select top 8 diverse views
assemble collage
save collage + metadata
```

Для оценки каждого вида программа строит простую маску объекта. Поскольку фон рендера белый, объект отделяется от фона через threshold: пиксели, которые заметно темнее белого, считаются частью объекта. На основе этой маски рассчитываются несколько признаков.

Используемые признаки:

```text
1. area_ratio — доля изображения, занятая объектом;
2. area_score — нормализованная оценка видимой площади объекта;
3. bbox_fill_score — насколько плотно объект заполняет свой 2D bounding box;
4. edge_ratio — доля контурных переходов в изображении;
5. edge_score — нормализованная оценка сложности силуэта и контуров;
6. base_score — итоговая визуальная оценка ракурса.
```

Итоговый score считается как взвешенная сумма:

```text
base_score =
    0.45 * area_score
  + 0.35 * edge_score
  + 0.20 * bbox_fill_score
```

После расчёта `base_score` программа выбирает восемь итоговых ракурсов. Выбор происходит не просто по top-8 максимальных оценок, а с учётом diversity. Это нужно, чтобы итоговый коллаж не состоял из почти одинаковых изображений. Первый вид выбирается по максимальному `base_score`, а следующие получают дополнительный бонус за угловое отличие от уже выбранных ракурсов.

Итоговый коллаж содержит восемь выбранных изображений. Каждый tile подписывается параметрами камеры и score:

```text
AZ000 EL35 S0.28
AZ180 EL35 S0.26
AZ270 EL35 S0.27
...
```

Где:

```text
AZ — azimuth angle;
EL — elevation angle;
S — base score выбранного вида.
```

На тестовой детали baseline выбрал преимущественно ракурсы с elevation 35° и 60°. Это показывает, что алгоритм отсекает почти боковые виды, где объект выглядит как тонкая полоска, и выбирает изображения, на которых лучше видны отверстия, толщина, силуэт и общая форма детали.

## CLI: запуск программы из терминала

Программа запускается через команду:

```bash

baseline-lab

```

Основные параметры:

```text

--demo              использовать встроенный demo-объект

--input             путь к пользовательской 3D-модели

--baseline          выбор baseline: iso или cap3d

--projection        тип ISO-проекции: first или third

--image-size        размер каждого изображения в пикселях

--output-root       корневая папка для сохранения результатов

--run-name          название конкретного запуска

--out               прямой путь для сохранения результата

```

Пример запуска ISO baseline:

```bash
baseline-lab --demo --baseline iso --projection third --image-size 384
```

Пример запуска Cap3D-style baseline:

```bash
baseline-lab --demo --baseline cap3d --image-size 384 --run-name demo_cap3d_views
```

Пример запуска Algorithmic Top Views baseline:

```bash
baseline-lab --input examples/models/test_part.obj --baseline algorithmic --image-size 384 --run-name test_part_algorithmic_top8
```

Пример запуска алгоритмического baseline на своей модели:

```bash
baseline-lab --input path/to/model.obj --baseline algorithmic --image-size 384 --run-name my_model_algorithmic_top8
```

Пример запуска на своей модели:

```bash
baseline-lab --input path/to/model.obj --baseline iso --projection third --image-size 384 --run-name my_model_iso
```

## Организация выходных данных

Для удобного просмотра результатов на GitHub программа сохраняет выходные данные каждого baseline в отдельную подписанную папку.

Общая структура сохранения:

```text

outputs/

    baseline_name/

        run_name/

            collage.png

            metadata.json

```

Например, результат ISO baseline сохраняется так:

```text

outputs/

    iso/

        demo_iso_third/

            collage.png

            metadata.json

```

А результат Cap3D-style baseline сохраняется так:

```text

outputs/

    cap3d/

        demo_cap3d_views/

            collage.png

            metadata.json

```

Результат Algorithmic Top Views baseline сохраняется так:

```text
outputs/
    algorithmic/
        test_part_algorithmic_top8/
            collage.png
            metadata.json
```

В папке `algorithmic` лежат результаты baseline, который автоматически выбирает наиболее информативные ракурсы из расширенного набора кандидатных видов.

Такой формат удобен для сравнения методов: можно открыть папку каждого baseline и сразу увидеть визуальный результат и параметры запуска. Это также упрощает публикацию результатов в GitHub-репозитории, потому что изображения и metadata-файлы лежат рядом и подписаны понятными именами.

## Metadata-файл

Пример metadata для ISO baseline:

Это позволяет проверить, почему алгоритм выбрал именно эти ракурсы, и делает метод интерпретируемым.

Metadata особенно важен для будущего режима сравнения baseline-методов. Когда появится несколько способов генерации коллажей, можно будет сравнивать не только картинки, но и параметры их создания: количество видов, тип камеры, время выполнения, размер изображений и другие характеристики.
